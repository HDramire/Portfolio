# IP Attorney Brief

## Project

`Portfolio` website for Henry Ramirez Jr.

## Current state

- The project is a public-facing static portfolio site.
- Remote repository currently points to:
  - `https://github.com/HDramire/Portfolio.git`
- The site includes custom HTML, CSS, JavaScript, written copy, and image assets.
- The repo now includes proprietary notices, a restrictive `LICENSE.md`, and crawler instructions in `robots.txt`.

## What I want to protect

- My original website code and content
- My custom visual presentation and written copy
- Any commercially important concept or differentiating idea connected to this project

## What I need advice on

1. Should I register copyright for the current site as one work or separate parts?
2. Are any images, resume content, or design assets better handled separately?
3. Does any part of the project raise trademark issues, such as a brand, name, or logo worth protecting?
4. Is there anything here that could qualify for patent protection, or is it too conceptual / publicly disclosed already?
5. What should remain private to preserve trade secret protection?
6. What public disclosures should I avoid before filing anything else?
7. If copying happens, what enforcement steps are realistic and cost-effective?

## Facts to tell the attorney

- Whether the site has already been publicly accessible
- Approximate first publication date
- Whether all code and assets were created by me
- Whether any contractor, collaborator, or AI tool contributed materially
- Whether the commercial value is in the code, brand, workflow, or business concept
- Whether I plan to license, sell, or commercialize the idea

## Materials to bring

- A ZIP of the current project files
- Screenshots of the site
- The current public URL, if live
- The GitHub repo URL
- A list of which images and text are fully original
- Any drafts or notes describing the underlying business idea

## Goal for the consultation

I want a practical recommendation on the right order of operations for:
- copyright registration
- keeping key differentiators private
- trademark, patent, or trade secret strategy
