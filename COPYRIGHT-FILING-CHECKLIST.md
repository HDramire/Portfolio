## Important limit

Copyright protects the expression of the work, not the underlying idea or business concept.

Official sources:
- https://www.copyright.gov/help/faq/faq-protect.html
- https://www.copyright.gov/registration/other-digital-content/
- https://www.copyright.gov/register/tx-programs.html

## Before you file

1. Confirm which assets are fully yours.
2. Remove any asset you do not own or do not want included in the filing.
3. Decide whether the project is unpublished or already published.
4. Save a dated copy of the exact version you want to register.
5. Keep your git history because it helps support authorship and creation dates.

## Suggested filing scope for this repo

Use one filing package for the current portfolio version if:
- you are the sole author, and
- the work is being registered as one cohesive website project.

If some images, resume content, or other materials were created at different times or by different people, review whether they should be excluded or registered separately.

## Filing steps

1. Go to the U.S. Copyright Office registration portal:
   https://www.copyright.gov/registration/
2. Use the online system.
3. Choose the application type that best matches the website/digital content.
4. Enter yourself as the author and claimant if that is accurate.
5. Upload the deposit copy.
6. Pay the filing fee.
7. Save the confirmation number and submitted files.

## Information you should have ready

- Legal name
- Mailing address
- Email address
- Project title
- Year completed
- First publication date, if already published
- Whether any part was previously published
- Whether any content was created by anyone else

## After filing

Keep these together:
- the confirmation email
- your uploaded deposit copy
- payment receipt
- screenshots of the submission
- git commit hash for the version you filed
